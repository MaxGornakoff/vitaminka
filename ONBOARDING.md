# Руководство по подключению магазина к Vitaminka Assistant

## Предварительные требования

- Развёрнутый backend Vitaminka (локально или на сервере)
- Секретный ключ администратора (`ADMIN_SECRET` из `.env`)
- JSON-фид товаров в формате Яндекс.Маркет (или совместимом)

---

## Шаг 1. Создать магазин в админке

1. Открыть [http://localhost:8000/admin](http://localhost:8000/admin)
2. Войти: логин и пароль из `.env` (`ADMIN_USERNAME` / `ADMIN_PASSWORD`)
3. Перейти в раздел **Магазины** → кнопка **Create**
4. Заполнить форму:

| Поле | Обязательное | Описание |
|------|:---:|---------|
| `shop_id` | ✅ | Уникальный идентификатор, латиница/цифры/дефис. Пример: `my-shop` |
| `name` | ✅ | Название магазина |
| `assistant_name` | — | Имя ассистента в заголовке виджета. По умолчанию: `Ассистент` |
| `domain` | — | Домен сайта. Пример: `myshop.ru` |
| `catalog_url` | — | URL JSON-фида (Яндекс.Маркет формат). Нужен для автосинка |
| `catalog_sync_interval_hours` | — | Интервал автообновления каталога в часах. Если не указан — берётся глобальный дефолт из `.env` (`CATALOG_SYNC_INTERVAL_HOURS=6`) |
| `manager_phone` | — | Телефон менеджера. Ассистент подставит его в ответе при вопросах о связи |
| `is_active` | — | Вкл/выкл магазин. По умолчанию включён |

5. Нажать **Save**

---

## Шаг 2. Получить API-ключ магазина

1. В списке магазинов кликнуть на нужный магазин → **View**
2. В разделе деталей скопировать поле **Api key** — это `X-API-Key` для последующих операций

> **Важно:** ключ автогенерируется при создании. Хранить в безопасном месте.

---

## Шаг 3. Загрузить каталог товаров

### Вариант A — через кнопку в админке (рекомендуется)

1. Открыть карточку магазина → **View**
2. В поле `Shop id` нажать кнопку **Синхронизировать каталог по catalog_url**
3. Убедиться, что `catalog_url` заполнен — ассистент скачает фид и проиндексирует товары

### Вариант B — через API

```bash
POST /api/shops/{shop_id}/catalog/sync
X-API-Key: <ваш api_key>
```

PowerShell:
```powershell
Invoke-RestMethod `
  -Method Post `
  -Uri "https://your-backend.com/api/shops/my-shop/catalog/sync" `
  -Headers @{"X-API-Key" = "ваш-api-key"}
```

### Вариант C — ручная загрузка JSON

```powershell
$catalog = @{
    products = @(
        @{ name = "Товар 1"; price = 990; url = "https://myshop.ru/p/1"; description = "Описание" }
    )
    replace_existing = $true
} | ConvertTo-Json -Depth 3

Invoke-RestMethod `
  -Method Post `
  -Uri "https://your-backend.com/api/shops/my-shop/catalog" `
  -Headers @{"X-API-Key" = "ваш-api-key"} `
  -ContentType "application/json" `
  -Body $catalog
```

## Шаг 4. Проверить работу ассистента

Открыть demo-страницу с виджетом, указав `shop_id` нового магазина:

```html
<!-- В demo-standalone.html замените: -->
const SHOP_ID = 'my-shop';
```

Или через URL-параметр, если это поддерживается вашей конфигурацией.

Задать несколько тестовых вопросов:
- "Что у вас есть из [категория]?"
- "Сколько стоит [название товара]?"
- "Как с вами связаться?"

---

## Шаг 5. Встроить виджет на сайт

Добавить в `<head>` или перед `</body>` вашего сайта:

```html
<script>
  window.VITAMINKA_API_URL = 'https://your-backend.com';
  window.VITAMINKA_SHOP_ID = 'my-shop';
</script>
<script src="https://your-backend.com/widget/widget.js"></script>
```

> Если ещё не настроена отдача статики — виджет можно разместить на CDN или в папке `/static` бэкенда.

### Что будет отображаться

- Заголовок виджета: значение `assistant_name` из настроек магазина
- Ответы: на основе загруженного каталога товаров + LLM (Cohere)
- Ссылки на товары: кликабельные, открываются в новой вкладке
- Телефон менеджера: кликабельная ссылка `tel:`

---

## Шаг 6. Настроить автообновление каталога

Каталог обновляется автоматически по расписанию:

- Планировщик проверяет магазины **каждый час**
- Для каждого магазина используется **его собственный интервал** (`catalog_sync_interval_hours`)
- Если интервал не задан — берётся глобальный дефолт из `.env`

Для изменения интервала конкретного магазина: открыть карточку магазина → **Edit** → изменить `catalog_sync_interval_hours`.

Рекомендуемые значения:
- Быстро обновляемый ассортимент: `2–4` часа
- Стандартный магазин: `6–12` часов
- Редко меняющийся каталог: `24–48` часов

---

## Шаг 7. Обновить настройки магазина

Через API:
```bash
PATCH /api/shops/{shop_id}
X-API-Key: <ваш api_key>
Content-Type: application/json

{
  "assistant_name": "Алиса",
  "manager_phone": "+7 (999) 123-45-67",
  "catalog_sync_interval_hours": 12
}
```

Через админку: карточка магазина → **Edit**.

## Справочник: переменные окружения (.env)

| Переменная | Описание |
|---|---|
| `ADMIN_SECRET` | Секрет для регистрации нового магазина через API |
| `ADMIN_USERNAME` / `ADMIN_PASSWORD` | Логин/пароль входа в `/admin` |
| `COHERE_API_KEY` | API-ключ Cohere (LLM + embeddings) |
| `CATALOG_SYNC_INTERVAL_HOURS` | Глобальный дефолт интервала синхронизации |
| `DATABASE_URL` | Строка подключения PostgreSQL |
